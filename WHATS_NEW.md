# Что нового в проекте

## Версия 1.1.0 (2026-01-29) - Расширенный бизнес-анализ

### ✨ Основные изменения

Теперь AI извлекает из диалогов **дополнительную бизнес-информацию**:

1. **📅 Желаемые сроки** - когда клиент хочет получить результат
2. **💰 Бюджет/Стоимость** - какие у клиента ожидания по деньгам  
3. **✅ Основные требования** - что ОБЯЗАТЕЛЬНО должно быть в продукте

### 📊 Как это выглядит в отчёте

#### Новая секция: "Желаемые сроки и стоимость"
```
┌─────────────────────────────────────────────────────┐
│ Желаемые сроки:    Через два месяца, до инвентаризации
│ Бюджет/Стоимость:  Ограниченный бюджет, облачная модель
└─────────────────────────────────────────────────────┘
```

#### Новая секция: "Основные требования к продукту"
```
┌─────────────────────────────────────────────────────┐
│ Что точно должно быть в финальном продукте:
│ ▪ Учёт товаров в реальном времени
│ ▪ Работа со штрихкодами
│ ▪ Синхронизация между складами
│ ▪ Мобильное приложение
│ ▪ Интеграция с 1С
└─────────────────────────────────────────────────────┘
```

### 🎯 Зачем это нужно?

**Для менеджеров**: 
- Сразу видно дедлайн клиента
- Понятны бюджетные ограничения
- Не нужно перечитывать весь диалог

**Для проектной команды**:
- Чёткий список must-have требований
- Проще планировать работу
- Меньше недопонимания

**Для руководства**:
- Вся критичная информация в одном месте
- Удобно для принятия решений
- Структурированные данные для анализа

### 🚀 Как начать использовать

Ничего делать не нужно! Просто запустите как обычно:

```bash
python main.py --input your_transcript.txt
```

AI автоматически найдёт и добавит новую информацию в отчёт.

### 💡 Что если информации нет?

Если в диалоге не упомянуты сроки/бюджет/требования - ничего страшного:
- Секции просто не появятся в отчёте
- Остальная информация будет как обычно
- Никаких ошибок не будет

### 🔄 Обновить старые отчёты?

Хотите пересоздать старые отчёты с новыми полями?

```bash
# Удалите кэш и запустите заново
python main.py --input old_transcript.txt --no-cache
```
